# GOLC-HOME-lab-climate-control

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

Отдельный репозиторий для климатического проекта на ESPHome.

Текущая реализация в этом репозитории посвящена управлению кондиционером Midea через ESPHome и Home Assistant по UART.

Лицензия проекта: `MIT`. Подробности см. в файле `LICENSE`.

## Зачем отдельный репозиторий

Этот репозиторий предназначен для быстрой реализации климат-контроля над кондиционером. В данном случае речь идёт о кондиционере Midea.

Он сделан потому, что в интернете по этой теме много разрозненной информации: часть материалов повторяется, часть противоречит друг другу, а часть оказывается неполной или устаревшей.

Здесь собрана рабочая на текущий момент информация и инструкция по реализации. Это не выдуманная с нуля схема, а собранный и сведённый в одно место практический материал из разных источников, на основе которого подготовлен единый рабочий код для климат-контроля.

Для чего это нужно:

- быстро получить рабочую основу для запуска климат-контроля
- не тратить время на разбор противоречивых инструкций из разных источников
- держать проверенный конфиг, пояснения и примеры в одном месте

## О чём этот проект

Это не generic-термостат ESPHome на `platform: thermostat`.

Это проект управления кондиционером Midea через:

- ESPHome
- `platform: midea`
- UART-подключение
- Home Assistant

## Быстрый старт

1. Скопировать `examples/climat-komnata.yaml` в свой ESPHome-проект.
2. Создать `secrets.yaml` по примеру `examples/secrets.example.yaml`.
3. Проверить реальные пины UART и соответствие платы.
4. Проверить, что для UART не конфликтует `logger`.
5. Загрузить прошивку и проверить появление climate-сущности в Home Assistant.

Перед первой проверкой подключения также стоит открыть `docs/wiring-and-troubleshooting.md` и отдельно сверить `TX/RX`, `GND` и конфликт `logger` с аппаратным UART.

## Важное замечание

В документации ESPHome есть общий раздел `climate` и отдельный раздел `thermostat`, но текущий рабочий проект использует именно `platform: midea`.

Поэтому основная логика и инструкция в этом репозитории привязаны к Midea UART-интеграции, а не к самостоятельной термостатической логике.

## Подключение UART и типовые причины, почему проект может не работать с первого раза

Для UART-подключения нужно помнить три базовых правила:

- `TX` одной стороны подключается к `RX` другой стороны
- `RX` одной стороны подключается к `TX` другой стороны
- `GND` должен быть общим

Если перепутать `TX/RX`, не подключить общий `GND` или оставить конфликтующий `logger`, проект может определяться в ESPHome, но кондиционер не будет нормально отвечать на команды.

Правильная схема подключения:

- `ESP TX -> Midea RX`
- `ESP RX -> Midea TX`
- `ESP GND -> Midea GND`

Неправильная схема подключения:

- `TX -> TX`
- `RX -> RX`

Что проверить в первую очередь:

1. Проверить, не перепутаны ли `TX` и `RX`.
2. Проверить, есть ли общий `GND`.
3. Проверить, что в `uart` выставлен `baud_rate: 9600`.
4. Проверить, не конфликтует ли `logger` с аппаратным UART.
5. Проверить, совпадает ли уровень сигнала, и нужен ли преобразователь `3.3V -> 5V`.

Если `TX` и `RX` подключены к аппаратным UART-пинам ESP, нужно проверить, не конфликтует ли с ними стандартный логгер.

В этом проекте используется:

```yaml
logger:
  baud_rate: 0
```

Практическое правило:

- если используется аппаратный UART на тех же пинах, ставить `baud_rate: 0`
- если конфликта нет и используется другой способ связи, блок можно не менять

Типовые симптомы и вероятные причины:

- устройство в ESPHome есть, но кондиционер не реагирует: проверить `TX/RX`, `GND`, `baud_rate: 9600`, конфликт `logger`, уровень сигнала
- прошивка загружается, но связь нестабильная: проверить провода, общий `GND`, питание ESP, преобразователь уровней
- команды проходят не всегда: проверить `timeout`, `num_attempts`, стабильность UART-линии и корректность подключения

Для расширенной схемы и более подробной диагностики см. `docs/wiring-and-troubleshooting.md`.
